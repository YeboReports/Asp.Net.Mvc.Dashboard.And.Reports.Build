YeboReports is an easy to use business intelligence (BI) that let you create dashboard, reports and documents.

The tool is written in ASP.NET MVC and it is a web application that runs on your premises. It is a self-hosted BI Dashboard reporting tool that makes it easier to create and share dashboards and reports.

Dashboards


  • PERMISSION BASED DASHBOARD. You can group your dashboard by user department, so that Finance will only see Finance data. Marketing will only see Marketing data. Manangement will only see Dashboards that are related to Manangement.etc
  • DASHBOARD CONTROL TYPES. Our dashboard supports: Table, Cards, Charts, Gauge, Pivot, Indicator, Progress, Images, Texts and more
  • END USER DASHBOARD DESIGN. End users can be allowed to design dashboards.
  • INTERACTIVE DASHBOARD. Our dashboard is interactive, you can sort, filter, drill down, export and do more
  • SUPPORTED DATA SOURCES. Our dashboards supports these datasources: Firebird MsSQL (Microsoft SQL), MySQL, MS Access, VistaDB, ODBC, Oracle, PostgreSQL, SQLCE, SQLite, XML, Json,CSV, Excel, Teradata and more..
  • DOWNLOAD DATA. Dashboard can be exported as an Image, Html, Excel and PDF.
  • NO PROGRAMMING KNOWLEDGE REQUIRED. You need a basic SQL knowledge so you can write queries to featch data from your database.
  • INSTANT DASHBOARD UPDATES. All changes on the dashboard design reflects to the users as soon as you click a Save button. No code deploy is required.
  • DASHBOARD INTERGRATION. We can add our dashboards to your existing application as well. Ask how?

Reports


  • PERMISSION BASED REPORTS. You can group your reports by user department, so that Finance will only see Finance data. Marketing will only see Marketing data. Manangement will only see Dashboards that are related to Manangement.etc
  • INTERACTIVE REPORTS. Our reports are interactive, you can sort, filter, accept user input, export and more
  • DOWNLOAD DATA. Based on the permission, the data can be extracted into a CSV, XLS, PDF and other formats.
  • INSTANT REPORT UPDATES. All changes on the reports reflects to the users as soon as you click a Save button. No code deploy is required.
  • GRID TYPES. You can easily create a normal a table grid and sub grid report, we support vertical report.
  • NO PROGRAMMING KNOWLEDGE REQUIRED. You need a basic SQL knowledge so you can write queries to featch data from your database. Once the setup is done, the endusers can then design the dashboard which is a drag and drop.
  • REPORTS INTERGRATION. We can add our reports to your existing Application. Ask how?

Invoices and Documents


  • GENERATES DOCUMENTS. The sytem can generate documents like, Sales summary report, Accounting Balance sheet report, invoices, invoice schedule, letters to the clients...etc
  • END USER DESIGN. There is an end user design for designing documents

YeboReports, Dashboard and Reports Security


  • LOGIN. Users need to login to use the system. Without a login you can not access the site.
  • END USER DESIGN. End user design is permission based. The admin will asign the users that are allowed to edit the dashboards.
  • DASHBOARD PERMISSION. You will only see dashboards that are asigned to your usergroup. A single dashboard can be assigned to multiple user groups.
  • REPORT PERMISSION. You will only see reports that are asigned to your usergroup. A single report can be assigned to multiple user groups.
  • DASHBOARD SHARING. You can also send the dashboard as a link, You can specify how many times a link can be viewed. You can also specify the expiry date of the link. A shared dashboard is dynamic which means that when the data changes on the server, the dashboard changes as well.
  • CREATING USERS. A temp user can be created to access dashboard and reports. To create a temp user all you need to do is to set the account expiry date. There is no limit on the number of users you can create. You can create users across organisations, which means your system can be used by workers, customers, suppliers, stackholders..etc Since everything is permission based users will only see dashboards and reports that are assigned to them.
